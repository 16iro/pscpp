# BOJ 11724 — 연결 요소의 개수

> **링크**: https://www.acmicpc.net/problem/11724 **티어**: Silver II **태그**:
> 그래프 이론, 그래프 탐색, 너비 우선 탐색, 깊이 우선 탐색 **제출 횟수**: 1회
> 만에 AC

## 접근 방법

### 핵심 아이디어

2차원 간선 그래프를 순회하며 방문하지 않은 간선을 루트로 BFS 탐색

### 시도한 접근들

1. **attempt #1** — 2차원 그래프 구성, BFS탐색으로 문제 풀이 구현 → AC

## 풀이

이 문제는 Connected Component의 개수를 구하는 문제다. 탐색되지 않은 정점에
대하여 해당 정점과 이어진 다른 모든 정점을 탐색한 다음, 나머지 정점 중 아직
탐색되지 않은 정점을 찾는 것이 기본 골자이다. 따라서 어떤 정점이 다른정점과
이어져있는지를 확인하는 것이 가장 중요하다고 판단하여 DFS/BFS를 먼저 생각해냈다.
메모리 제한은 512M이고, N이 최대 1000, M이 최대 499,500까지이므로 512MB 안에서
2차원 배열까지는 문제없이 표현 가능하다고 판단. 문제풀이의 직관성을 위해 그래프
배열은 2차원으로 설정하였다. 무방향 그래프이므로, 배열은 전체의 절반만 사용한다.
따라서 입력 간선 중 u의 값이 v보다 무조건 작거나 같게 설정하여 I행렬의 상단
부분만 데이터가 점유하도록 제어한다. 이후 그래프의 탐색 범위를 좁히기 위해서
최소값,최대값을 알아낸 이후 해당 범위 내에서 BFS를 시작한다. BFS를 탐색할 때,
graph[u][v]에 대해서 각 정점에 연결된 간선을 알아내는 방법은 해당 정점
graph[u][u]를 기준으로 위, 오른쪽으로 탐색하여 graph[min~u][u],
graph[u][u~max]를 순회하며 방문하지 않은 간선을 알아내여 큐에 추가한다. v도
마찬가지로 진행한다. 이때 조심해야하는 것이 N-M이 1보다 큰 경우, 아무도 방문하지
않는 정점이 무조건 하나 이상은 발생하기 때문에 해당 정점을 연결 요소에
반영해준다.

### 알고리즘 / 자료구조

- 2차원 그래프 / BFS 탐색
- 시간복잡도: O(n^3) => 2차원 배열을 순회하며 각 정점에 대해 BFS를 선형순회함,
  3초 제한을 통과하는 이유는 잘 모르겠음.
- 공간복잡도: O(n^2) => 2차원 배열 사용, int[1000][1000] = 4,000,000 byte이므로
  512mb 제한을 충분히 충족

### 정확성

코드 자체는 정확하게 동작하여 AC를 받았다. 그러나 README에서 "방문하지 않은
간선을 루트로 BFS 탐색"이라고 했는데, 일반적인 BFS는 정점을 단위로 탐색한다.
실제 코드도 큐에 `pair<int,int>`(간선)을 넣는 간선 기반 BFS를 구현하고 있다.
이것이 의도적인 설계인지 README에서 명확히 드러나지 않는다.

### 효율성

이 문제에서 반복적으로 하는 질의는 "정점 X에 인접한 정점은 무엇인가?"이다.
인접 행렬에서 이 질의 비용은 O(N), 인접 리스트에서는 O(degree(X))이다. 표준
정점 기반 BFS + 인접 리스트를 사용하면 전체 복잡도가 O(N+M)으로 떨어진다.
현재 구현은 O(N² + M·N)으로, N=1000에서 최악의 경우 ~5×10⁸ 연산에 달해
3초 제한에서 위험하다. 또한 `graph[10001][10001]`은 N ≤ 1000인 문제에서
약 381MB를 점유하여 MLE 경계에 놓여 있다.

### 안정성

- `min`, `max`, `count`를 변수명으로 사용하고 있는데, `using namespace std`
  하에서 `std::min`, `std::max`, `std::count`와 이름이 충돌할 수 있다.
- M=0 케이스는 `vertex.empty()` 분기로 올바르게 처리하고 있다.
- 배열 크기 10001은 N ≤ 1000 제약 대비 과도하며 MLE 위험이 있다.

### 가독성

- BFS 큐에 `pair<int,int>`(간선)이 들어가는 비표준 구조로, 코드를 처음 읽는
  사람이 파악하기 어렵다.
- `range`를 `pair<int,int>`로 전달하는 것보다 `int lo, int hi`가 더 직관적이다.
- BFS 내부의 u/v 각각에 대한 상삼각/하삼각 4중 루프 구조가 복잡하다. 정점
  기반 BFS라면 단일 루프로 충분하다.

### 총평

문제를 정확하게 풀어 AC를 받았고, 고립 정점 처리 등 예외 상황도 잘 고려했다.
그러나 이 문제의 본질은 "정점에 인접한 정점을 빠르게 찾는 것"이며, 이 질의에
최적화된 자료구조는 인접 리스트이다. 인접 행렬 + 간선 기반 BFS 조합은
동작하지만 O(N+M)을 O(N³)으로 끌어올리는 원인이 되었다. 다음 단계로 정점
기반 탐색과 인접 리스트 표현을 연습해보는 것을 권장한다.
