# BOJ 10830 — 행렬 제곱

> **링크**: https://www.acmicpc.net/problem/10830
> **티어**: Gold IV
> **태그**: 수학, 분할 정복, 분할 정복을 이용한 거듭제곱, 선형대수학
> **제출 횟수**: 2회 만에 AC

---

## 접근 방법

- 시간 제한 : 1초
- 메모리 제한 : 256MB

2by2 ~ 5by5 행렬을 100,000,000,000제곱한다고 가정해보자.
만약 B가 POT 넘버일 경우, 같은 행렬에 대해서는 결합법칙이 항상 성립하기 때문에 분할 계산기 가능함. 즉, log_2^POT번만 곱하면 된다.
그러나 B가 NPOT 넘버일 경우, 이를 POT의 합으로 분리한 다음, 행렬 연산에 반영해야한다.
B가 7일 경우 4 + 2 + 1로 분할하여 A^4 x A^2 x A 로 곱한다.
이때 B를 구성하는 모든 작은 수가 POT이므로, 최대 POT행렬을 구하는 과정에서 이전 POT행렬을 캐싱하는 방식으로 계산 생략이 가능하다.
따라서 NPOT넘버인 B의 최소연산량은 (int)(log_2^B)+(B의 POT구성개수)
B의 POT구성개수는 2진법으로 전개했을 때 1비트인 자리수 개수와 같다.

모든 행렬 연산이 효율적으로 이루어진다고 가정했을때 100,000,000,000를 log2로 취하면 36.541이 나온다. 최악의 경우 연산량은 약 73회 이내가 되므로 1초 안에 계산이 가능하다.

## 풀이

N와 B를 입력받은 후, 비트연산자를 사용하여 B의 POT요소를 분리한다.
POTbit중 가장 큰 값을 가져와서 POTmatrix에서 거듭제곱연산을 미리 캐싱한다.
이후 최종 값을 연산할 행렬배열 matrix에서 [0]번째 배열을 미리 아이젠밸류화한 다음, POTset(작은 값부터 정렬되어있음!)을 순회하면서 각 POTmatrix에 캐싱된 배열을 곱해준다.

### 정확성
README에 기술된 풀이 의도(B를 이진 분해 → POT 행렬 캐싱 → 순차 곱셈)가 코드에 올바르게 반영되어 있다. `NPOT2POT`에서 비트를 추출하고, `powMatrix`에서 A^(2^k)를 반복 제곱으로 구하며, 최종 루프에서 해당 POT 행렬들을 곱하는 흐름이 정확하다. 다만 입력 원소가 정확히 1000일 수 있는데, `POTmatrix[1]`에 모듈러 없이 원본을 저장하는 점은 의도 명확성 측면에서 개선 여지가 있다.

### 효율성
접근 방식 자체는 올바르나, 구현이 필요 이상으로 복잡하다. 표준 반복적 거듭제곱(iterative binary exponentiation)은 B를 미리 분해할 필요 없이, 최하위 비트 검사 → 결과에 곱셈 → 밑 제곱 → B 우시프트를 반복하면 된다. 이 방식은 `set<int>` POTset이 불필요하고, 중간 POT 행렬을 모두 저장할 필요 없어 O(N²logB) → O(N²) 공간으로 줄일 수 있다.

### 안정성
배열 범위(`[50]` vs maxbit ≤ 37)와 오버플로우(`uint64_t` 사용)는 안전하다. 마지막 줄 뒤에 개행을 출력하지 않는 점은 BOJ에서 보통 문제없지만, 관례상 넣는 것이 안전하다.

### 가독성
`// eigenvalued` 주석은 단위행렬(identity matrix) 초기화를 의미하는데, eigenvalue는 다른 선형대수 개념(고유값)이므로 용어를 수정할 필요가 있다. 변수명 `ny`, `nx`, `n`은 `row`, `col`, `k` 등이 더 직관적이다.

**총평:** 분할 정복을 이용한 행렬 거듭제곱의 핵심 원리를 정확히 이해하고, 본인만의 용어로 풀어낸 점이 좋다. 표준 iterative binary exponentiation 패턴을 익히면 코드량과 공간 사용을 크게 줄일 수 있고, README에서는 "핵심 아이디어" 플레이스홀더 작성과 용어 정확성(eigenvalue → identity matrix)을 보완할 필요가 있다.
