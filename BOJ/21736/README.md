# BOJ 21736 — 헌내기는 친구가 필요해

> **링크**: https://www.acmicpc.net/problem/21736
> **티어**: Silver II
> **태그**: 그래프 이론, 그래프 탐색, 너비 우선 탐색, 깊이 우선 탐색, 격자 그래프
> **제출 횟수**: {N}회 만에 AC

---

## 접근 방법

### 핵심 아이디어

격자 그래프 기반으로 BFS를 통해 I를 루트 포인트로 시작하여 P를 몇번 만나는지 측정한다.

### 시도한 접근들

1. **attempt #1** — 최초구현 → WA
2. **attempt #2** — 2차원 배열 초기화 → SEGFAULT
2. **attempt #3** — dy,dx 더할 때 맵 범위 clamp처리 → AC

---

## 풀이

I를 시작점으로 하여 그래프를 탐색하며 도달가능한 P의 개수를 구하는 문제이므로, 그래프를 넓게 탐색해야한다. 따라서 BFS를 사용한다.
입력을 그대로 활용하여 2차원 인접행렬 기반의 탐색이 가능하므로 자료구조는 2차원 배열을 사용한다.

### 알고리즘 / 자료구조

- 격자 그래프
- 시간복잡도: O(N^2)인접행렬
- 공간복잡도: O(N^2)

## 코드 리뷰 (powered by AI)

> README 평가: **16/30** (로직 6 · 복잡도 5 · 구현 5)

### 정확성
README의 "BFS로 I에서 시작하여 P를 카운트"라는 의도는 코드에 올바르게 반영되어 AC를 받았다. 다만 배열 초기화 `{'X',}`는 `graph[0][0]`만 `'X'`로 설정하고 나머지는 `'\0'`이 되어 의도와 실제 동작이 다르며, `cell = 'O'`(line 25)는 지역 변수만 변경하고 `graph[nx][ny]`는 변경하지 않아 코드 의도가 불명확하다.

### 효율성
BFS O(NM) 탐색은 적절하다. 다만 `clamp`를 경계 체크 대용으로 쓰면 경계에서 자기 자신 셀을 불필요하게 재검사하게 된다. 범위를 벗어나면 `continue`하는 방식이 의미적으로 정확하고 불필요한 검사도 없앤다. 또한 P→O→X 2단계 분기 대신 `cell == 'X'`이면 `continue`, `cell == 'P'`이면 카운트, 공통 마킹+push라는 단일 흐름이 더 간결하다.

### 안정성
`rn`, `rm`이 초기화 없이 선언되어 `'I'`가 입력에 없으면 미초기화 변수로 BFS를 호출할 수 있다(문제 조건상 보장되지만 컴파일러 경고 대상). `clamp` 사용은 논리적 오류를 은폐할 수 있는 구조로, 방문 마킹 로직에 버그가 있을 경우 무한 루프 위험이 있다.

### 가독성
`rn`/`rm`은 의미가 불분명하고, 변수명 `queue`가 `std::queue`와 동일하여 혼동 여지가 있다. `cell` 지역 변수를 수정하는 패턴이 의도 파악을 어렵게 만든다. 경쟁 프로그래밍 코드로서 길이는 적절하나, 위 세 가지가 코드 이해를 방해한다.

**총평:** BFS 격자 탐색이라는 정공법으로 잘 풀었으나, `clamp`를 경계 체크 대용으로 쓴 점과 배열 초기화 문법(`= {'X',}`)이 의도와 다르게 동작한다는 점을 인지할 필요가 있다. README는 핵심 아이디어는 전달하고 있으나, 복잡도 표기의 정확성(O(N^2) → O(NM))과 알고리즘 선택 근거의 깊이를 보강하면 더 좋겠다.
